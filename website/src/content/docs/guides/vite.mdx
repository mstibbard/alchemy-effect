This guide walks you through deploying a Vite-based project to
Cloudflare — whether it's a static site, an SSR framework, or a
single-page application.

<Steps>

1. **Set up your Vite project**

   Create a project with your framework of choice (TanStack Start,
   SolidStart, Nuxt, vanilla Vite, etc.).

   :::tip
   You don't need to install or configure the Cloudflare Vite plugin
   yourself. Alchemy automatically injects the plugin at build time
   so your `vite.config.ts` stays framework-standard — no Cloudflare-
   specific config needed.
   :::

2. **Create the Stack**

   Create an `alchemy.run.ts` with `Cloudflare.Vite`. No build
   command, output directory, or Wrangler config needed — Alchemy
   handles it all via the Vite plugin.

   ```typescript
   // alchemy.run.ts
   import * as Alchemy from "alchemy-effect";
   import * as Cloudflare from "alchemy-effect/Cloudflare";
   import * as Effect from "effect/Effect";

   export default Alchemy.Stack(
     "MyApp",
     { providers: Cloudflare.providers() },
     Effect.gen(function* () {
       const app = yield* Cloudflare.Vite("App", {
         compatibility: { flags: ["nodejs_compat"] },
       });
       return { url: app.url };
     }),
   );
   ```

   Most SSR frameworks need `nodejs_compat` for Node.js API access.
   Pure static sites can omit it.

3. **Configure framework-specific options (optional)**

   SolidStart benefits from `runWorkerFirst` so the server handles
   routing before falling back to static assets:

   ```typescript
   yield* Cloudflare.Vite("SolidStart", {
     compatibility: { flags: ["nodejs_compat"] },
     assets: {
       config: { runWorkerFirst: true },
     },
   });
   ```

   For SPAs (Vue, React, etc.), configure client-side routing:

   ```typescript
   yield* Cloudflare.Vite("Vue", {
     compatibility: { flags: ["nodejs_compat"] },
     assets: {
       config: {
         htmlHandling: "auto-trailing-slash",
         notFoundHandling: "single-page-application",
       },
     },
   });
   ```

4. **Configure memoization (optional)**

   Alchemy content-hashes your input files to decide whether
   `vite build` needs to re-run. By default it hashes every
   non-gitignored file plus the nearest lockfile.

   For large projects, narrow the scope with `memo.include` so only
   relevant files trigger rebuilds:

   ```typescript
   yield* Cloudflare.Vite("Docs", {
     memo: {
       include: ["src/**", "content/**", "package.json"],
     },
   });
   ```

   Set `lockfile: false` if your build output doesn't depend on
   installed packages.

5. **Deploy**

   ```sh
   alchemy deploy
   ```
