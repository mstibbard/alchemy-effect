This guide walks you through creating and deploying an AWS Lambda
Function with alchemy-effect.

:::note
For the plain async handler pattern, see the [Async Lambda guide](/guides/async-lambda).
For the full API reference, see the [Lambda Function provider](/providers/aws/lambda/function).
:::

<Steps>

1. **Define the Function class**

   Create a file for your Function. The `main` prop tells Alchemy
   which file to bundle and deploy.

   ```typescript
   // src/JobStorage.ts
   import * as Context from "effect/Context";
   import * as Effect from "effect/Effect";

   export class JobStorage extends Context.Service<
     JobStorage,
     {
       putJob(job: Job): Effect.Effect<Job, PutJobError>;
       getJob(jobId: string): Effect.Effect<Job | undefined, GetJobError>;
     }
   >()("JobStorage") {}
   ```

2. **Create resources**

   Inside the Layer that implements `JobStorage`, create the resources
   you need. Resources are scoped to the Layer — they're created at
   deploy time and available for binding.

   ```typescript
   import * as DynamoDB from "alchemy/AWS/DynamoDB";
   import * as SQS from "alchemy/AWS/SQS";

   export const JobStorageDynamoDB = Layer.effect(
     JobStorage,
     Effect.gen(function* () {
       const table = yield* DynamoDB.Table("JobsTable", {
         partitionKey: "id",
         attributes: { id: "S" },
       });
       const queue = yield* SQS.Queue("JobsQueue");
       // ...
     }),
   );
   ```

3. **Bind capabilities**

   Each AWS binding maps 1:1 with an IAM action. You compose exactly
   the permissions your function needs — nothing more.

   ```typescript
   const getItem = yield * DynamoDB.GetItem.bind(table);
   const putItem = yield * DynamoDB.PutItem.bind(table);
   ```

   `getItem` is a typed function: pass a `Key`, get back a
   `GetItemOutput`. At deploy time, the Policy attaches
   `dynamodb:GetItem` on `table.tableArn` to the Lambda's IAM role.

4. **Add a sink**

   Bind an SQS sink for batched writes. The sink handles chunking
   messages into `SendMessageBatch` calls automatically.

   ```typescript
   const sink = yield * SQS.QueueSink.bind(queue);
   ```

   You'll pipe records into this sink from the event source below.

5. **Subscribe to an event source**

   Use `DynamoDB.stream()` to subscribe to change records from the
   table. The `.process()` callback receives a typed `Stream` of
   records that you can transform and drain into the sink.

   ```typescript
   yield *
     DynamoDB.stream(table, {
       streamViewType: "NEW_AND_OLD_IMAGES",
       startingPosition: "LATEST",
       batchSize: 10,
     }).process((stream) =>
       stream.pipe(
         Stream.map((record) =>
           JSON.stringify({
             eventName: record.eventName,
             keys: record.dynamodb.Keys,
             newImage: record.dynamodb.NewImage,
           }),
         ),
         Stream.run(sink),
       ),
     );
   ```

   At deploy time this enables DynamoDB Streams on the table, grants
   stream read IAM, and creates a `Lambda.EventSourceMapping`. At
   runtime, Lambda invokes the function with stream records and the
   event source Layer pipes them into your `Stream`.

6. **Implement the service**

   Wire the bound capabilities into the service methods:

   ```typescript
   const putJob = (job: Job) =>
     putItem({
       Item: {
         id: { S: job.id },
         content: { S: job.content },
       },
     }).pipe(
       Effect.map(() => job),
       Effect.catchCause((cause) =>
         Effect.fail(
           new PutJobError({
             message: `Failed to store job "${job.id}": ${cause}`,
             cause,
           }),
         ),
       ),
     );

   return JobStorage.of({ putJob, getJob });
   ```

7. **Compose Layers**

   Provide the binding Layers and event source Layers on the
   `JobStorage` Layer. This is where you declare the full set of
   runtime capabilities and background processes.

   ```typescript
   export const JobStorageDynamoDB = Layer.provideMerge(
     Layer.effect(
       JobStorage,
       Effect.gen(function* () {
         // ... resource creation, bindings, event source, service impl
       }),
     ),
     Layer.mergeAll(Lambda.TableEventSource, SQS.QueueSinkLive).pipe(
       Layer.provideMerge(
         Layer.mergeAll(
           DynamoDB.GetItemLive,
           DynamoDB.PutItemLive,
           SQS.SendMessageBatchLive,
         ),
       ),
     ),
   );
   ```

   The Layers split into two groups:
   - **Capability layers** (`GetItemLive`, `PutItemLive`,
     `SendMessageBatchLive`) — runtime SDK wrappers
   - **Event source / sink layers** (`Lambda.TableEventSource`,
     `SQS.QueueSinkLive`) — background process wiring

8. **Define the Function**

   Create the Lambda Function class. Provide the `JobStorage` Layer
   (and any other service Layers) via `Effect.provide`.

   ```typescript
   // src/JobFunction.ts
   import * as AWS from "alchemy/AWS";
   import * as Effect from "effect/Effect";
   import * as HttpServerResponse from "effect/unstable/http/HttpServerResponse";

   export default class Api extends AWS.Lambda.Function<Api>()(
     "Api",
     { main: import.meta.filename, url: true },
     Effect.gen(function* () {
       return {
         fetch: Effect.succeed(HttpServerResponse.text("Hello")),
       };
     }),
   ) {}
   ```

2. **Add resource bindings**

   Create resources and bind capabilities in the init phase. They
   become available to all runtime handlers.

   ```typescript
   import * as DynamoDB from "alchemy-effect/AWS/DynamoDB";

   Effect.gen(function* () {
     // init — create resources and bind capabilities
     const table = yield* DynamoDB.Table("UsersTable", {
       partitionKey: "pk",
       attributes: { pk: "S" },
     });
     const getItem = yield* DynamoDB.GetItem.bind(table);
     const putItem = yield* DynamoDB.PutItem.bind(table);

     return {
       // runtime — use them
       fetch: Effect.gen(function* () {
         yield* putItem({
           Item: { pk: { S: "user#1" }, name: { S: "Alice" } },
         });
         const result = yield* getItem({
           Key: { pk: { S: "user#1" } },
         });
         return yield* HttpServerResponse.json(result.Item);
       }),
     };
   });
   ```

3. **Create the Stack**

   Wire the Function into your `alchemy.run.ts` stack file.

   ```typescript
   // alchemy.run.ts
   import * as Alchemy from "alchemy";
   import * as AWS from "alchemy/AWS";
   import * as Effect from "effect/Effect";
   import Api from "./src/Api.ts";

   export default Alchemy.Stack(
     "MyApp",
     { providers: AWS.providers() },
     Effect.gen(function* () {
       const api = yield* Api;
       return { url: api.functionUrl };
     }),
   );
   ```

4. **Deploy**

   ```sh
   alchemy deploy
   ```

</Steps>
