This guide walks you through creating and deploying a Cloudflare Worker
with Effect-native bindings.

:::note
For the plain async handler pattern, see the [Async Worker](/guides/async/worker) guide.
For the full API reference, see the [Worker provider](/providers/cloudflare/worker).
:::

<Steps>

1. **Define the Worker class**

   Create a file for your Worker. The `main` prop tells Alchemy which
   file to bundle and deploy.

   ```typescript
   // src/Api.ts
   import * as Cloudflare from "alchemy/Cloudflare";
   import * as Effect from "effect/Effect";
   import * as HttpServerResponse from "effect/unstable/http/HttpServerResponse";

   export default class Api extends Cloudflare.Worker<Api>()(
     "Api",
     { main: import.meta.path },
     Effect.gen(function* () {
       return {
         fetch: Effect.succeed(HttpServerResponse.text("Hello")),
       };
     }),
   ) {}
   ```

2. **Create the Stack**

   Wire the Worker into your `alchemy.run.ts` stack file.

   ```typescript
   // alchemy.run.ts
   import * as Alchemy from "alchemy";
   import * as Cloudflare from "alchemy/Cloudflare";
   import * as Effect from "effect/Effect";
   import Api from "./src/Api.ts";

   export default Alchemy.Stack(
     "MyApp",
     { providers: Cloudflare.providers() },
     Effect.gen(function* () {
       const api = yield* Api;
       return { url: api.url };
     }),
   );
   ```

3. **Deploy**

   ```sh
   alchemy deploy
   ```

</Steps>

## Binding another Worker (RPC)

Workers can call each other's methods with typed RPC. Bind a sibling
Worker in the init phase and call its methods at runtime:

```typescript
// src/Api.ts
import WorkerB from "./WorkerB.ts";

export default class Api extends Cloudflare.Worker<Api>()(
  "Api",
  { main: import.meta.path },
  Effect.gen(function* () {
    // init — bind the other Worker
    const b = yield* Cloudflare.Worker.bind(WorkerB);

    return {
      // runtime — call its RPC methods
      fetch: Effect.gen(function* () {
        const greeting = yield* b.greet("world");
        return HttpServerResponse.text(greeting);
      }),
    };
  }),
) {}
```

Importing WorkerB's class (its Tag) doesn't pull in WorkerB's runtime
code — the bundler tree-shakes the `.make()` implementation. See the
[Dynamic Workers](/guides/effect/dynamic-workers) guide for
bi-directional binding.

## Adding resource bindings

Resources like R2, D1, KV, and Durable Objects are bound in the init
phase and used at runtime. Each has its own guide:

- [R2 Bucket](/guides/effect/r2-bucket) — object storage with
  Effect bindings
- [D1 Database](/guides/effect/d1-database) — serverless SQLite with
  Effect bindings
- [KV Namespace](/guides/effect/kv-namespace) — key-value storage
  with Effect bindings
- [Durable Object](/guides/effect/durable-object) — stateful objects
  with RPC, WebSocket hibernation, and Effect bindings
